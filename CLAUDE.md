# Finance Daily Digest

Automated pipeline: fetch real-time finance headlines → summarize via Claude API → store to Google Drive and/or locally. Runs on a daily cron or GitHub Actions schedule.

## Commands

```bash
npm run dev          # ts-node src/index.ts — run full pipeline once
npm run auth         # ts-node src/auth.ts — one-time Google OAuth flow
npm run build        # tsc — compile to dist/
npm run start        # node dist/index.js — run compiled output
```

Add to `package.json` scripts:
```json
"scripts": {
  "dev": "ts-node src/index.ts",
  "auth": "ts-node src/auth.ts",
  "build": "tsc",
  "start": "node dist/index.js"
}
```

## Architecture

```
src/index.ts       Entry point — orchestrates the pipeline
src/config.ts      Loads and validates .env, exports typed config
src/fetch.ts       Calls Claude API with web_search tool, returns markdown report
src/store.ts       Writes report locally to reports/ and uploads to Google Drive
src/auth.ts        One-time OAuth2 flow to generate token.json (run manually)
reports/           Local report output — git-ignored
logs/              Cron log output — git-ignored
```

## Environment

All config lives in `.env` — never commit it. Required vars:

```
ANTHROPIC_API_KEY          Anthropic API key
GOOGLE_CLIENT_ID           Google OAuth client ID
GOOGLE_CLIENT_SECRET       Google OAuth client secret
GOOGLE_REDIRECT_URI        http://localhost:3000/oauth2callback
GOOGLE_DRIVE_FOLDER_ID     (optional) target Drive folder ID
TOPICS                     Comma-separated list e.g. Markets,Macro economy,Tech stocks
REPORT_DEPTH               brief | standard | deep
CUSTOM_FOCUS               (optional) extra focus hint e.g. "Federal Reserve, NVDA"
```

`token.json` is generated by `npm run auth` and must not be committed. For GitHub Actions, store its contents as secret `GOOGLE_TOKEN_JSON`.

## Key implementation notes

- Claude API call uses `web_search_20250305` tool — do not remove it; the pipeline depends on real-time search results
- Model is pinned to `claude-sonnet-4-20250514` — do not change without testing token budget
- `max_tokens` is set to 2000; increase to 4000 only for `depth=deep` with long `TOPICS` lists
- Drive upload is non-fatal — pipeline continues with a warning if Drive auth fails
- `token.json` needs a refresh handler for long-running production deployments (see Phase 5 in implementation plan)

## Google Drive setup

1. Enable Drive API at [console.cloud.google.com](https://console.cloud.google.com)
2. Create OAuth 2.0 Desktop client → copy credentials to `.env`
3. Run `npm run auth` once → generates `token.json`
4. Optionally set `GOOGLE_DRIVE_FOLDER_ID` to target a specific folder

## Scheduling

**Cron (local/VM)**:
```bash
0 7 * * * cd /path/to/finance-digest && npx ts-node src/index.ts >> logs/digest.log 2>&1
```

**GitHub Actions**: see `.github/workflows/daily-digest.yml` — triggered at `0 7 * * *` UTC. All `.env` vars must be added as repository secrets.

## Report output

- Filename format: `finance-digest-YYYY-MM-DD.md`
- Local path: `reports/finance-digest-YYYY-MM-DD.md`
- Drive: uploaded to root or `GOOGLE_DRIVE_FOLDER_ID` folder if set
- Format: structured markdown with Summary, Top Headlines, Key Themes, Market Signals (+ Macro Analysis and Key Risks for `depth=deep`)

## Common failures

| Symptom | Cause | Fix |
|---|---|---|
| `No text content returned` | Web search quota or model error | Check API key and model availability |
| `token not found` | `token.json` missing | Run `npm run auth` |
| `invalid_grant` on Drive | Token expired | Re-run `npm run auth` or add refresh handler |
| Empty `reports/` dir | Pipeline errored silently | Check `logs/digest.log` or run `npm run dev` manually |
| Drive upload skipped | Missing Drive env vars | Verify `GOOGLE_CLIENT_ID`, `GOOGLE_CLIENT_SECRET` in `.env` |
